# 프로젝트 개요

**우르르**는 AI 기술을 활용한 뷰티 공동구매 커머스 플랫폼입니다.
프로그래머스 생성형 AI 활용 백엔드 데브코스 1기 최종 프로젝트로, `함께 할수록 저렴해지는 공동구매`라는 컨셉으로 제작되었습니다.
FastAPI와 Faiss를 활용한 AI 기반 상품 추천 시스템, 공동구매 시스템, OAuth 소셜 로그인, 결제 처리 등 이커머스 비즈니스 로직을 처리하는 서비스를 제공합니다.

# 시스템 아키텍처 설계

![아키텍처 다이어그램](https://eaalkymxyfskjojh.public.blob.vercel-storage.com/portfolio/ururu/architecture.png)

### 프론트엔드 배포

프론트엔드는 Vercel에 배포해 정적 자산을 CDN으로 제공했습니다. 프론트 배포를 백엔드 릴리즈와 분리하여, UI 변경을 더 빠르게 반영하고 백엔드 배포 리스크에 영향을 받지 않도록 했습니다.

### 백엔드 인프라

백엔드 트래픽 흐름은 **Route 53 → ALB → Nginx → Spring Boot** 구조로 구성했습니다.  
애플리케이션 서버 앞단에 ALB와 Nginx를 두어 외부 진입 지점을 단순화하고, 리버스 프록시 계층에서 라우팅과 배포 전환을 제어할 수 있도록 설계했습니다.

Spring Boot 애플리케이션은 Docker 컨테이너 기반으로 운영했고, MySQL(RDS), Redis, FastAPI 추천 서비스를 분리해 각 워크로드의 역할을 나눴습니다.
### 배포 전략

배포는 **Nginx upstream 전환 기반의 Blue-Green 방식**으로 구성했습니다. 동일 서버 내에서 Blue/Green 애플리케이션 컨테이너를 분리 운영하고, 전환 시점에만 라우팅 대상을 변경하도록 설계해 서비스 중단을 최소화하고 문제 발생 시 이전 버전으로 빠르게 복귀할 수 있도록 했습니다.

### AI 서비스

AI 추천 및 벡터 기반 검색 기능은 FastAPI 서비스로 분리했습니다. Spring Boot 메인 API와 별도 프로세스·별도 서버로 분리함으로써, 의존성 충돌과 리소스 사용 패턴 차이로 인한 영향을 줄이고 서비스 경계를 명확히 했습니다.

### 모니터링

모니터링 스택은 별도 EC2에서 운영했습니다. Prometheus와 Grafana를 애플리케이션 서버와 분리해 배포나 애플리케이션 장애 상황에서도 운영 지표를 보다 안정적으로 수집·확인할 수 있도록 구성했습니다.

### AS-IS

5명의 백엔드 팀원이 협업하는 과정에서 코드 품질이 개인 역량에 의존하는 문제가 발생했습니다. 
리뷰어의 경험과 시간에 따라 리뷰 품질이 달라지고, 리뷰 부담이 특정 인원에게 집중되면서 
코드리뷰가 형식적으로 진행되는 경우가 있었습니다.


### TO-BE
