# 프로젝트 개요

**우르르**는 AI 기술을 활용한 뷰티 공동구매 커머스 플랫폼입니다.
프로그래머스 생성형 AI 활용 백엔드 데브코스 1기 최종 프로젝트로, `함께 할수록 저렴해지는 공동구매`라는 컨셉으로 제작되었습니다.
FastAPI와 Faiss를 활용한 AI 기반 상품 추천 시스템, 공동구매 시스템, OAuth 소셜 로그인, 결제 처리 등 이커머스 비즈니스 로직을 처리하는 서비스를 제공합니다.

# 시스템 아키텍처 설계

![아키텍처 다이어그램](https://eaalkymxyfskjojh.public.blob.vercel-storage.com/portfolio/ururu/architecture.png)

### 프론트엔드 배포

프론트엔드는 Vercel에 배포해 정적 자산을 CDN으로 제공했습니다. 프론트 배포를 백엔드 릴리즈와 분리하여, UI 변경을 더 빠르게 반영하고 백엔드 배포 리스크에 영향을 받지 않도록 했습니다.

### 백엔드 인프라

백엔드는 VPC 내부에서 인바운드를 **ALB → Nginx → Spring Boot**로 단일화했습니다. 외부 노출 지점을 최소화하고, 리버스 프록시 계층에서 라우팅·접근 제어·배포 전환을 일괄 통제할 수 있도록 구성했습니다. 
또한 ALB 헬스체크를 기반으로 비정상 인스턴스를 자동으로 제외해 장애 확산을 줄였습니다.

### 배포 전략

배포는 **Blue-Green 구조**로 구성했습니다. 배포 중에도 기존 인스턴스가 트래픽을 처리하도록 유지하면서, 전환 시점에만 스위칭하여 무중단 릴리즈와 즉시 롤백이 가능하도록 운영했습니다.

### AI 서비스 격리

AI 추천 및 벡터 검색은 별도 서비스로 격리했습니다. 라이브러리 의존성과 부하 특성이 다른 워크로드가 메인 API에 전파되지 않도록 구성했습니다.

### 모니터링

모니터링 스택을 독립적으로 구성하여 배포 및 장애 상황에서도 로그와 메트릭이 유실되지 않도록 했습니다. 운영 지표를 대시보드로 즉시 확인할 수 있게 구성했습니다.

# CodeRabbit AI 기반 코드리뷰 프로세스 구축
![코드래빗 리뷰](https://eaalkymxyfskjojh.public.blob.vercel-storage.com/portfolio/ururu/coderabbit_review_image.png)

### AS-IS

5명의 백엔드 팀원이 협업하는 과정에서 코드 품질이 개인 역량에 의존하는 문제가 발생했습니다. 
리뷰어의 경험과 시간에 따라 리뷰 품질이 달라지고, 리뷰 부담이 특정 인원에게 집중되면서 
코드리뷰가 형식적으로 진행되는 경우가 있었습니다.


### TO-BE

**CodeRabbit AI**를 코드리뷰 프로세스에 도입하여, `AI 자동 리뷰 → 피드백 반영 → 로테이션 기반 최종 리뷰`로 이어지는 협업 플로우를 정착시켰습니다.


### 결과
- PR마다 최소 품질 체크가 자동으로 선행되어 리뷰가 문법/스타일 지적보다 핵심 설계 논의와 합의에 집중될 수 있었습니다.
- 로테이션 리뷰로 리뷰 부담을 분산해 특정 인원 의존도를 낮췄고, 팀 전체가 코드 변경 맥락을 공유하는 속도가 개선되었습니다.
